# Unified 3-fold CNN splits

Drop-in CSVs for CNN single-label classification, **identical image-to-fold
mapping** as the VLM (PaliGemma2 / Qwen2.5-VL / etc.) experiments. Use these
to train CNNs whose results are directly comparable.

## Files

| file               | what                                                            |
|--------------------|-----------------------------------------------------------------|
| `fold_0.csv`       | fold 0: train + val + test (one row per image, `split` column)  |
| `fold_1.csv`       | fold 1                                                          |
| `fold_2.csv`       | fold 2                                                          |
| `holdout_final.csv`| final unseen test set — excluded from every fold                |
| `classes.csv`      | canonical class vocabulary (`class_id`, `name`, `n_total`)      |
| `summary.csv`      | row counts per (fold, split, source)                            |

## Split rule (per fold N ∈ {0,1,2})

  train = images whose unified split ∈ {fold_K : K != N}   (~60%)
  val   = images whose unified split == holdout_val          (~10%, **same in every fold**)
  test  = images whose unified split == fold_N               (~30%, rotates)
  holdout_final.csv  = unified split == holdout_test         (~10%, **never trained on, ever**)

The val set is constant across folds — it's a holdout, not a fold-internal slice.
The holdout_final set is reserved for one final comparison run after model
selection; do not use it for hyper-parameter tuning.

## Columns

  image_path   relative to the datasets root (see "Image storage" below)
  source       medgr | odin | vippstar
  dish         unified dish key (stable id; useful for sanity grouping)
  label        canonical PRIMARY label (lowercased) — use this for single-label CNN
  all_labels   comma-joined canonical labels (multi-food vippstar plates only)
  n_labels     1, 2, or 3 — filter `n_labels == 1` for clean single-label training
  group        leakage group id used during split generation (FYI only)
  split        train | val | test (within this fold's CSV)

## Class vocabulary

`classes.csv` contains 249 canonical classes (primary-label view).
Use `class_id` ↔ `name` for label encoding. The label space matches the VLM's
homogenised dish_name target so cross-model comparison is fair.

## Image storage

Images are stored under the datasets root with three different on-disk formats:

  medgr     `.npy` numpy arrays   (under `MedGR Dataset/...`)
  odin      `.npy` numpy arrays   (under `odin/...`)
  vippstar  `.jpg` JPEGs          (under `data/raw/...`)

Single loader that handles both formats:

```python
import numpy as np
from PIL import Image, ImageFile
ImageFile.LOAD_TRUNCATED_IMAGES = True  # a few JPEGs in the wild are truncated

def load_image(path):
    p = str(path)
    if p.endswith(".npy"):
        arr = np.load(p)
        if arr.dtype != np.uint8:
            arr = (arr * 255).clip(0, 255).astype(np.uint8) if arr.max() <= 1.0 else arr.astype(np.uint8)
        return Image.fromarray(arr).convert("RGB")
    img = Image.open(p); img.load()
    return img.convert("RGB")
```

## Reproducibility

These splits were generated by `scripts/regenerate_unified_splits.py` using
group-aware StratifiedGroupKFold with seed=42 over per-source group keys:

  medgr     <dish>/bucket_{N//10}
  odin      <dish>/plate_{P}
  vippstar  <dish_folder>/<portion>

Stratum = `<source>/<dish>`; all 249 strata are represented in
every fold AND in both holdouts.

Generated by `scripts/export_cnn_splits.py` from `unified_splits.csv`.
